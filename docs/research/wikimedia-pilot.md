## 推荐的官方 API 流程

### 1. 作品身份与中文条目

- Wikidata 搜索：`https://www.wikidata.org/w/api.php?action=wbsearchentities&search={query}&language=zh&format=json`
- 已知 QID 后取实体：`https://www.wikidata.org/wiki/Special:EntityData/{QID}.json`
- 中文维基批量取简介、条目属性及 URL：

```text
https://zh.wikipedia.org/w/api.php
  ?action=query
  &prop=extracts|pageprops|info|revisions
  &exintro=1
  &explaintext=1
  &inprop=url
  &rvprop=ids|timestamp
  &redirects=1
  &titles={title1|title2|...}
  &format=json
  &formatversion=2
```

`pageprops.wikibase_item` 是条目对应的 QID；`revisions[0].revid` 与时间戳应一并写入本地快照。模块的官方自说明可由 [Action API help](https://www.mediawiki.org/wiki/API:Main_page)、[`query+extracts` help](https://zh.wikipedia.org/w/api.php?action=help&modules=query%2Bextracts)、[`query+pageprops` help](https://zh.wikipedia.org/w/api.php?action=help&modules=query%2Bpageprops) 和 [`wbgetentities` help](https://www.wikidata.org/w/api.php?action=help&modules=wbgetentities) 查阅。

`wbsearchentities` 只应生成候选，随后用类型、年份与创作者人工消歧，再以 `wbgetentities` 固定 QID 和 `sitelinks.zhwiki.title`。Wikidata 的[官方版权页](https://www.wikidata.org/wiki/Wikidata:Copyright)说明其主命名空间等结构化数据采用 CC0，可将 QID、声明和 sitelink 存入本地数据。TextExtracts 建议导入约 600–1000 字原始引言后再人工压缩；其[官方文档](https://www.mediawiki.org/wiki/Extension:TextExtracts#API)不推荐依赖 `exsentences`，因为缩写等会造成错误断句。

### 2. 图片许可核验

对候选文件所在的 wiki 调用 `imageinfo`，不要只看图片 URL：

```text
https://commons.wikimedia.org/w/api.php
  ?action=query
  &prop=imageinfo
  &iiprop=url|extmetadata
  &titles=File:{filename}
  &format=json
  &formatversion=2
```

官方模块说明：[API:Imageinfo](https://www.mediawiki.org/wiki/API:Imageinfo)。至少保存：`descriptionurl`、`Artist`、`Credit`、`LicenseShortName`、`LicenseUrl`、`UsageTerms`、`AttributionRequired`、`Copyrighted`、`NonFree`、核验日期。

自动判定建议：

1. `NonFree=true` 或许可为 `Fair use / 合理使用`：直接标记 `unavailable`，不得下载到本站。
2. 文件来自 Commons 且许可在允许清单内：标记 `review`，人工核对文件页、作者、许可、来源链及可能的商标／人格权限制后才能变成 `approved`。
3. 没有机器可读许可、来源不完整或许可相互矛盾：标记 `review`，不能自动发布。
4. API 应使用说明性 User-Agent、批量请求、缓存、退避和限速；本次核验中 Action API 曾返回 `You are making too many requests`，说明导入器必须处理 `429/限流`，参见 [MediaWiki API etiquette](https://www.mediawiki.org/wiki/API:Etiquette)。

## 文本许可与署名要求

- Wikimedia Foundation [Terms of Use §7](https://foundation.wikimedia.org/wiki/Policy:Terms_of_Use/en#7._Licensing_of_Content) 规定，维基项目文本通常在 CC BY-SA 下提供（除非另有标注）；再利用者需要署名，并在修改或衍生时遵守相同方式共享等条件。
- 本站不应把压缩后的简介表述为原创。每件作品详情建议固定显示：`内容改编自维基百科，CC BY-SA 4.0`、原条目链接、修订号／抓取日期，以及“已压缩和改写”的变更说明。
- 按条款，可通过链接到被复用的条目完成作者归属；修改后的文本须采用 CC BY-SA 4.0 或更新版本，清楚说明已修改，并链接许可证。应仅把简介字段标为 CC BY-SA，而不是误称整个网站都采用该许可。
- 署名入口应清楚可见；全站“来源与许可”页可汇总完整记录，但不能让单件作品完全失去可追溯性。
- 图片没有统一的“维基许可”。每个文件都必须依据其文件页单独判断。Commons 的官方站外复用说明见 [Commons:Reusing content outside Wikimedia](https://commons.wikimedia.org/wiki/Commons:Reusing_content_outside_Wikimedia)；中文维基的非自由文件只是在其特定条目语境下主张合理使用，不构成本站的再授权。

## 实施门槛

试点实现应满足以下条件后再扩展到全部 62 件：

1. 简介导入器保存原文、修订号、抓取日期和编辑稿，编辑稿需人工批准。
2. 图片导入器默认拒绝；只有 `approved` 才能渲染真实封面。
3. 详情页与独立许可页都能追溯到原条目／文件页。
4. 构建测试拒绝缺少许可字段的 `approved` 图片，也拒绝 `NonFree=true` 图片进入发布资源。
5. 本轮唯一可先接入的真实图片是 Persepolis 的 Commons 文件；其余 7 件继续使用统一占位图。
